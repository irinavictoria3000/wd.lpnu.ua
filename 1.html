Соціальні мережі

1. ШІ-асистенти та основи промпт-інжинірингу

Мета роботи

Навчитися формулювати ефективні запити (системні, Few-Shot, Chain-of-Thought) для аналізу бізнес-ніші, збору трендів та формування первинної архітектури сайту. Опанувати синергію різних ШІ-моделей (ChatGPT, Claude, Grok), використовуючи їхні сильні сторони на різних етапах аналітики.

Стек інструментів для роботи

У 2026 році між трьома провідними платформами штучного інтелекту — ChatGPT, Claude та Grok — сформувалися чіткі відмінності в системній філософії, функціональних можливостях та підходах до верифікації даних.
  • ChatGPT (OpenAI) спроектований як соціально приємний та лояльний асистент. Модель схильна підтверджувати думки користувача, хвалити за хороші запитання та підлаштовуватися під заданий контекст, що робить її ідеальною для творчості та мозкових штурмів. ChatGPT використовує пошуковий індекс Bing/OpenAI
  • Grok (xAI) базується на пошуку об’єктивних фактів. На відміну від ChatGPT, Grok може прямо вказувати користувачеві на його помилки або логічні суперечності та надавати критичну оцінку. Grok має унікальну перевагу — прямий доступ до бази даних платформи X (Twitter) в реальному часі, що допомагає йому аналізувати тренди та події, які відбулися буквально кілька хвилин тому.
  • Claude (Anthropic) займає проміжну позицію, пропонуючи строгий академічний та аналітичний інструментарій. Модель фокусується на безпеці, точності обробки великих масивів тексту та написанні коду. Claude більше покладається на внутрішню документацію користувача через конектори (Google Drive, Microsoft 365).

В 2026 році ChatGPT допомагає автоматизувати роботу, трансформуючись з простого чат-бота на багатофункціонального асистента, який здатний заощаджувати години робочого часу щотижня за рахунок генерації контенту, написання коду та аналітики.

Автоматизація створення та адаптації контенту

ChatGPT виступає як креативний партнер, що автоматизує повний цикл роботи з текстами.

  • Генерація ідей та планування. Модель допомагає проводити брейнштормінг тем для блогів, сценаріїв для YouTube та розробляти контент-календарі.
  • Написання чернеток. автоматичне створення перших варіантів електронних листів, звітів, описів продуктів та публікацій для соціальних мереж за лічені секунди.
  • Переробка контенту. ChatGPT може автоматично перетворити один довгий допис в блозі на серію коротких постів для LinkedIn, сценарій відео або текст для розсилки, зберігаючи єдиний голос бренду.
  • Оптимізація. Модель автоматизує перевірку граматики, стилістичне редагування та підбір SEO-ключів для покращення видимості контенту.
Технічна автоматизація та розробка

Для фахівців технічних спеціальностей ChatGPT пропонує інструменти для прискорення розробки.

  • Codex. Спеціалізований модуль для написання та налагодження програмного коду на таких мовах, як Python, HTML, PHP та інших.
  • Робота через API. Розробники можуть інтегрувати можливості ChatGPT у власні додатки для автоматизації сумаризації відгуків, класифікації даних за тональністю, перекладу та виправлення помилок у великих масивах тексту.
  • Тестування. ChatGPT здатний автоматично генерувати тести для коду та пояснювати складні фрагменти логіки.
Організація робочих процесів та аналітика

Платформа пропонує інструменти для структурування щоденної рутини.

  • Проєкти (Projects). Надають можливість організовувати чати в тематичні папки (наприклад, «Маркетинг» або «Розробка»), що допомагає тримати всі матеріали за конкретним робочим напрямком в одному місці.
  • Deep Research. Інструмент для глибокого дослідження ринку, розрахунку прибутковості бізнес-ідей та аналізу продуктів.
  • Інтеграція додатків (Apps). ChatGPT можна підключати до сторонніх інструментів, таких як Photoshop або сервіси продуктивності, для створення безшовних робочих ланцюжків.
  • Командна робота. Функція групових чатів надає можливість кільком користувачам працювати над одним завданням разом з ШІ.

Ефективність автоматизації в ChatGPT значно зростає, якщо використовувати формулу якісного промпту, яка містить визначення Ролі (хто є ШІ), Завдання (що зробити), Деталей (контекст) та Цілі (очікуваний результат).

В 2026 році Grok від xAI пропонує потужний набір інструментів для автоматизації — від планування повторюваних завдань до глибокої аналітики даних та інтеграції в зовнішні бізнес-процеси.

Модуль керування завданнями (Grok Tasks)

Це ключова функція для автоматизації рутини, яка допомагає налаштовувати заплановані та повторювані промпти.

  • Регулярні звіти. Можна налаштувати Grok на щотижневий збір оновлень у вибраній індустрії або щоденний моніторинг новин через мережу X та вебпошук.
  • Автоматичні розсилки. Створення кастомних чернеток для розсилок або дайджестів за заданим розкладом (щодня, щотижня, щомісяця чи щороку).
  • Фонові процеси. Завдання виконуються автономно, а Grok надсилає сповіщення про готовність результатів.
Аналітична автоматизація та робота з кодом

Grok має вбудовані інструменти, які діють як «віртуальний аналітик».

  • Code Interpreter (Виконання коду). Модель здатна самостійно писати та запускати скрипти на Python прямо в чаті. Це надає можливість автоматизувати аналіз великих масивів даних (наприклад, файлів CSV), розраховувати складні математичні показники та будувати графіки або діаграми на основі наданих даних.
  • Структурований вивід (JSON). Grok може повертати дані в чистому форматі JSON без зайвого тексту. Це критично важливо для передачі результатів роботи ШІ в інші системи, такі як Zapier, n8n, AirTable або власні скрипти автоматизації.
Агентські ланцюжки та спеціалізація ролей

Для автоматизації складних досліджень Grok надає можливість використовувати концепцію агентських ланцюжків, де в одному промпті можна оркеструвати роботу «команди» ролей

  • Дослідник. Автоматизує пошук у реальному часі через X та веб.
  • Логік. Перевіряє математичні розрахунки та логіку коду.
  • Критик. Надає критичну оцінку та шукає слабкі місця в плані чи стратегії. Можна задати послідовність дій (пошук → аналіз → критика → синтез), і Grok виконає весь ланцюжок за один прохід.
Організація контексту через Проєкти та Робочі простори

Автоматизація підготовки контексту реалізована через структурування знань.

  • Проєкти (Projects). Надають можливість групувати пов’язані чати, файли (PDF, Word, Excel, код) та інструкції в одному місці. Кожен новий чат в проєкті автоматично успадковує цей контекст, що позбавляє необхідності щоразу надавати вступну інформацію.
  • Вбудована пам'ять (Memory). Grok може запам'ятовувати уподобання користувача, проєктні деталі та попередні обговорення між різними сесіями, роблячи взаємодію більш персоналізованою без повторень.
Творча автоматизація (Imagine та Vision)
  • Мультимодальний аналіз. Через камеру або завантажені скріншоти Grok може автоматично інтерпретувати помилки на екрані, аналізувати дизайни конкурентів або пояснювати складні діаграми.
  • Генерація відео. Модуль Imagine надає можливість створювати довгі відео (до кількох хвилин), автоматизуючи підтримку візуальної узгодженості персонажів та фонів між окремими сценами.

Завдяки прямому доступу до бази даних мережі X в реальному часі, Grok забезпечує найвищий рівень автоматизації для завдань, пов'язаних з моніторингом трендів, перевіркою фактів та швидким реагуванням на події в світі.

В 2026 році Claude допомагає автоматизувати роботу через перетворення простого чат-інтерфейсу на структуровану екосистему, де Проєкти (Projects) та Навички (Skills) виступають ключовими елементами управління знаннями та процесами.

Автоматизація контексту через Проєкти (Projects)

Проєкти надає можливість створити виділене робоче середовище для конкретних завдань, що автоматизує підготовку ШІ до роботи.

  • Єдине джерело істини (Knowledge Base). В проєкт можна завантажувати документи, PDF-файли, електронні таблиці або код, які стають базою знань для всіх подальших діалогів у межах цієї папки. Це позбавляє необхідності щоразу повторно завантажувати файли або нагадувати ШІ деталі поточного проєкту.
  • Проєктні інструкції. Для кожного проєкту можна задати кастомні інструкції щодо тону, стилю бренду чи специфічних правил обробки даних. Наприклад, можна наказати Claude завжди відповідати в форматі звітності або використовувати специфічні терміни обраної компанії.
  • Групування знань. Проєкти автоматично організовують історію чатів, артефакти та файли в одному місці, що допомагає швидко повертатися до роботи над складними довгостроковими завданнями, такими як запуск продукту чи фінансовий аудит.
Автоматизація процесів через Навички (Skills)

Навички — це багаторазові шаблони робочих процесів або «картки з рецептами», які надають Claude можливість виконувати складні послідовності дій автоматично.

  • Багаторазові алгоритми. Користувач може один раз навчити Claude певному процесу (наприклад, перевірці коду на вразливості або звірці транзакцій) і зберегти його як навичку. Наступного разу виконання цього складного завдання тригерується автоматично через текстовий запит або команди з косою рискою (/commands).
  • Стандартизація виводу. Навички забезпечують стабільність результату, оскільки Claude вже має вбудовану логіку валідації та необхідний формат вихідних даних для конкретної ролі (наприклад, фінансового аналітика).
  • Кастомні навички. Можна створювати власні навички, наприклад «Навички Студента», щоб ШІ завжди писав відповіді клієнтам саме вашим корпоративним голосом без додаткових пояснень у кожному чаті.
Синергія з іншими інструментами автоматизації

Для повної автоматизації Claude поєднує Проєкти та Навички з технічними можливостями платформи.

  • Заплановані завдання (Scheduled Tasks). Надають можливість налаштувати Claude на автоматичний запуск аналізу даних щопонеділка або після оновлення файлів в проєктній папці.
  • Конектори (Connectors). Підключають Claude безпосередньо до Google Drive, Microsoft 365 чи Airtable, що допомагає навичкам працювати з «живими» даними без ручного завантаження.
  • Артефакти (Artifacts). Виділені вікна для редагування результатів роботи (сайтів, коду, планів) в реальному часі надають можливість миттєво вносити зміни в автоматично згенерований контент.

Інженерія запитів

Промпт-інжиніринг — це дисципліна розробки та оптимізації інструкцій (промптів) для ефективного використання великих мовних моделей (LLM) в різних прикладних завданнях. В 2026 році цей напрям все частіше називають контекст-інжинірингом, оскільки фокус змістився з пошуку «магічних фраз» на архітектурне керування всім станом системи: інструкціями, історією діалогу, зовнішніми даними та інструментами.

Основні принципи якісного промпту

Якісний промпт базується на чотирьох «китах»: ясності, контексті, обмеженнях та ітеративному вдосконаленні. Замість розмитих запитів слід використовувати конкретні команди, пам’ятаючи про принцип «сміття на вході — сміття на виході».

Для створення ідеального запиту фахівці рекомендують використовувати формулу успішного промпту.

  • Роль. Ким має бути ШІ (наприклад, «дій як фінансовий аналітик»).
  • Завдання. Що саме потрібно зробити («напиши звіт»).
  • Деталі. Додатковий контекст, цільова аудиторія та стиль.
  • Ціль. Очікуваний результат або формат виводу (наприклад, JSON або таблиця).

Головна порада для початківців. Ставтеся до ШІ як до здібного виконавця — давайте чіткі інструкції, не змушуйте його вгадувати ваші думки та завжди перевіряйте фінальний результат.

Щоб підвищити якість відповідей моделі, використовують перевірені стратегії.

  • Навчання на прикладах (Few-shot Prompting). Додавання 2–5 канонічних прикладів в промпт допомагає моделі краще зрозуміти структуру та тон відповіді. Дослідження показують, що 5 прикладів забезпечують понад 90% точності відтворення стилю бренду.
  • Ланцюжок міркувань (Chain of Thought). Прохання до ШІ «думати крок за кроком» змушує модель спочатку вибудувати логічну логіку, а потім надавати відповідь. Це критично для математичних та логічних завдань.
  • Негативні обмеження (Negative Prompts). Чіткі вказівки щодо того, чого не слід робити (наприклад, «не використовуй жаргон», «не додавай пояснень»), допомагають уникнути зайвого тексту.
  • Використання роздільників. Застосування XML-тегів (наприклад, ...) або потрійних лапок допомагає моделі чітко розрізнити інструкції від вхідних даних.
Налаштування тональності та стилю

Тон повідомлення визначає, як аудиторія сприйматиме контент. Промпт-інжиніринг надає можливість гнучко налаштовувати тональні профілі.

  • Офіційний (Formal). Для бізнес-звітів без емоджі та особистих займенників.
  • Емпатичний (Empathetic). Для підтримки клієнтів, з акцентом на розумінні емоцій.
  • Переконливий (Persuasive). Для маркетингу, з використанням психологічних тригерів та закликів до дії.
Технічні параметри

Крім самого тексту, на результат впливають технічні налаштування моделі.

  • Temperature (Температура). Низькі значення (0.1–0.2) роблять відповіді стабільними та детермінованими (добре для коду), високі (0.8+) — творчими та різноманітними.
  • Max Tokens. Обмежує довжину відповіді для контролю лаконічності та витрат.
Від промптів до контекст-інжинірингу

В сучасних системах промпт-інженерія перетворилася на управління контекстним вікном. Оскільки моделі можуть втрачати увагу при великих обсягах даних (ефект «контекстного зносу»), використовуються такі методи.

  • Своєчасне отримання (Just-in-Time Retrieval). Завантаження лише необхідної інформації в момент запиту.
  • Компактизація (Compaction). Автоматичне стиснення історії діалогу для збереження найважливіших деталей.
  • Агентські архітектури. Розподіл складного завдання між кількома спеціалізованими суб-агентами (планувальник, виконавець, критик).

Матриця варіантів (бізнес-тематики на вибір студента)

Кожен студент обирає для себе варіант. Тема буде наскрізною для всіх наступних лабораторних робіт. Обирайте ту тематику, яка вам справді цікава. Якщо ви любите ігри — беріть "Dice&Card", якщо спорт — "Upgrade Your Body". Коли ви завантажите ці дані у свій workflow в ЛР №2, ШІ-асистенти допоможуть вам створити такий контент і логіку, які змусять цей сайт виглядати як реальний стартап, готовий до виходу на ринок.

  1. Сайт-візитка ветеринарної клініки "ЗооВет". Специфікація: калькулятор вартості операцій та вакцинації.
  2. Сайт-платформа для локального бренду одягу "БрендUa". Специфікація: віртуальний помічник підбору розміру одягу.
  3. Портал кіберспортивної академії "КіберЛьвів". Специфікація: калькулятор аналізу ігрового часу та розрахунку плану тренувань.
  4. Лендинг сервісу розумного планування бюджету "Потік монет". Специфікація: інтерактивний симулятор заощаджень/повзунки.
  5. Сайт розплідника та готелю для тварин "Готель Лапка". Специфікація: форма-калькулятор розрахунку вартості проживання улюбленця.
  6. Платформа еко-туризму "Зелені стежки Карпат". Специфікація: інтерактивний калькулятор вартості еко-туру з вибором кількості днів, гідів, прокату спорядження та трансферу.
  7. Лендинг для EdTech-курсу "ШІ для підлітків". Специфікація: конвертер тривалості та вартості навчання залежно від обраних модулів із системою персональних знижок за промокодами.
  8. Сайт-портфоліо архітектурного бюро "Новий простір". Специфікація: інтерактивний слайдер-порівняння "До/Після" реалізованих інтер'єрів та форма розрахунку вартості дизайну за квадратний метр.
  9. Вебдодаток для крафтової кав'ярні "Смажена кава". Специфікація: конструктор власного кавового міксу з вибором сорту зерна, ступеня обсмаження, пакування та інтеграцією онлайн-кошика.
  10. Портал мікрозелені та сіті-фермерства "Мікрозелень". Специфікація: калькулятор розрахунку щоденної норми суперфудів залежно від ваги, віку та цілей здоров'я користувача.
  11. Платформа підбору та оренди ретро-автомобілів "Ретро Авто". Специфікація: інтерактивний календар-калькулятор вартості оренди з урахуванням погодинного або добового тарифу та страховки.
  12. Сайт фітнес-кемпу "Онови своє тіло". Специфікація: ШІ-трекер калорій та калькулятор індексу маси тіла (ІМТ/BMR) з автоматичною генерацією базового спортивного меню).
  13. Сервіс підписки на бокси настільних ігор "Кубики&Карти". Специфікація: інтерактивний тест-опитувальник на JS для автоматичного підбору ідельної настільної гри на вечір за параметрами компанії.
  14. Смарт-платформа для психологічної допомоги "Спокій розуму". Специфікація: інтерактивний трекер емоційного стану — користувач обирає настрій, а скрипт динамічно пропонує дихальну вправу або аудіо-релакс.
  15. Каталог крафтових сирів та вина "Гурман-Інфо". Специфікація: система інтерактивної фільтрації та підбору ідеальної гастрономічної пари "сир + вино" за регіоном походження та смаковими профілями

Покроковий хід виконання роботи

Етап 1. Налаштування робочого середовища та запуск Рольової моделі (Всі ШІ-сервіси)
  1. Авторизуватися в інтерфейсах ChatGPT, Claude та Grok. Використовувати корпоративний (від Політехніки) обліковий запис.
  2. Створити окрему тематичну гілку чату або робочу область під назвою майбутнього проєкту. Опція Проєкти в лівій панелі інферфейсу даних сервісів.
  3. Надіслати в кожен сервіс Системний промпт, який ініціалізує роль Старшого бізнес-аналітика.
    • Шаблон Промпту №1 (System/Role Prompt). "Ти — Senior Business Analyst & UI/UX Expert з 15-річним досвідом розробки цифрових продуктів. Твоє завдання — допомогти мені спроектувати архітектуру та логіку для вебсайту за темою [ВСТАВИТИ НАЗВУ ТА ОПИС СВОГО ВАРІАНТА]. Твої відповіді мають бути структурованими, технічно грамотними, без «води», в форматі професійних артефактів (таблиці, списки, markdown). Чи зрозуміла тобі твоя роль та специфікація проєкту?"

*Markdown – це легка мова розмітки, яку можна використовувати для додавання елементів форматування до текстових документів.

Етап 2. Аналіз актуальних трендів ринку в реальному часі (Робота з Grok)

Оскільки тренди у вебі змінюються щодня, варто використовувати Grok з активованим веб-пошуком (або пошуком по мережі X), щоб знайти найсвіжіші фішки для обраної ніші.

  1. Написати запит для аналізу конкурентів та технологічних новинок в обраній сфері за поточний рік.
  2. Шаблон Промпту №2 (Grok Real-Time Search).
    • "Проаналізуй ринок [назва ніші, наприклад: фінтех-лендинги або сайти готелів для тварин] за останні 6-12 місяців. Які головні візуальні та функціональні тренди зараз домінують в світі та Україні? Які інтерактивні функції очікують побачити користувачі на такому сайті? Виділи 5 ключових інсайтів."
  3. Результат з Grok скопіювати для фінального звіту та перенести в ChatGPT/Claude як контекст для наступних кроків.
Етап 3. Глибокий аналіз цільової аудиторії та виявлення потреб користувачів (Робота з Claude)

Claude найкраще справляється зі складним логічним аналізом психології користувачів та структуруванням великих текстів.

  1. Використовуючи техніку Ланцюжок Думок (Chain-of-Thought, CoT), змусити ШІ виявити реальні проблеми потенційних клієнтів майбутнього сайту.
  2. Шаблон Промпту №3 (Chain-of-Thought в Claude).
    • "На основі раніше визначених трендів давай проаналізуємо цільову аудиторію нашого сайту. Думай крок за кроком (Chain-of-Thought): Крок 1 — Виділи 2 основні сегменти аудиторії. Крок 2 — Для кожного сегмента визнач 3 головні потреби (проблеми, страхи, незручності), з якими вони стикаються в інтернеті під час пошуку таких послуг. Крок 3 — Запропонуй, як саме наш майбутній сайт (завдяки інтерфейсу чи функціям) може закрити ці потреби. Оформи результат у вигляді чіткої Markdown-таблиці."

*Таблиці Markdown використовують символи вертикальної риски (|) та дефіси (-) для створення структурованих табличних даних у текстових документах.

Етап 4. Розробка специфікації та сценаріїв користувача User Stories (Робота з ChatGPT або Claude)

Тепер переходимо до технічного проектування. Користувацька історія (User Story) описує функціонал сайту з точки зору кінцевого споживача за класичною формулою: "Як [роль користувача], я хочу [дія], щоб [мета]".

  1. Застосуйте техніку Few-Shot Prompting, надавши ШІ приклад правильної структури.
  2. Шаблон Промпту №4 (Few-Shot в ChatGPT/Claude)
    • "Грунтуючись на потребах цільової аудиторії, розроби набір технічних User Stories для нашого сайту. Мені потрібні історії як для звичайного відвідувача, так й для користувача, який взаємодіє з нашим головним інтерактивним віджетом [вказати назву віджета з варіанта, наприклад: калькулятором заощаджень]. Надаю приклад правильного формату (Few-Shot):
      • Як новий відвідувач, я хочу бачити чіткий калькулятор вартості на головній сторінці, щоб миттєво дізнатися ціну без дзвінків менеджеру.
      • Як власник кота, я хочу вибрати тип кімнати у формі, щоб побачити фінальну суму за добу проживання моєї тварини.
      Напиши щонайменше 7-8 детальних User Stories, які покривають: навігацію, роботу з інтерактивним віджетом та форму зворотного зв'язку."

Коли ви виконуватимете лабораторні роботи, ніколи не ставтеся до написання User Stories як до формальності чи нудної писанини для викладача. Поставте себе на місце звичайної людини, яка зайшла на ваш сайт через мобільний 4G-інтернет, стоячи в черзі чи в метро. Сформулюйте її бажання, закрийте її потреби — і ви створите веб-ресурс, який принесе реальний прибуток бізнесу та зробить вас зіркою в будь-якій ІТ-компанії.

User Story (Історія користувача) — це дуже коротке, просте й зрозуміле опис конкретної функції сайту чи додатка, написане з точки зору кінцевого користувача (а не програміста). Замість того, щоб описувати, що має робити система, User Story описує, яку проблему людина хоче вирішити за допомогою вашого веб-ресурсу.

Класична формула User Story

У всьому світі розробники (і ШІ-асистенти, якщо їм надіслати правильний системний промпт) використовують єдину формулу для написання таких історій, яка миттєво відповідає на три головні запитання: Хто? Що? Навіщо?

Як [тип/роль користувача],
я хочу [виконати певну дію/отримати функцію],
щоб [досягти конкретної мети / вирішити свою проблему].

  • Застарілий підхід (як писали раніше)
    • «На сторінці сайту має бути присутній калькулятор з JavaScript-обробником подій onchange та полями вводу для розрахунку вартості оренди автомобіля».
    • Тут є інструкція "що зробити", але немає розуміння, чи зручно це людині й навіщо це їй
  • Сучасний підхід (User Story)
    • «Як потенційний клієнт сервісу оренди ретро-автомобілів, я хочу наочно пересувати повзунок кількості годин оренди в ШІ-віджеті, щоб миттєво бачити фінальну вартість зі страховкою та не витрачати час на дзвінки менеджерам».
    • Цей варіант відразу дає дизайнеру та програмісту розуміння: віджет має бути візуальним (повзунки), реактивним (миттєвий прорахунок без перезавантаження) і закривати потребу клієнта (небажання телефонувати).
Навіщо User Stories потрібні майбутньому спеціалісту у 2026 році?

Це ідеальний міст між вами та ШІ. Якщо ви прийдете до Claude чи ChatGPT і скажете: "Напиши мені код для сайту кав'ярні", він видасть вам стандартний, шаблонний код. Але якщо ви згодуєте йому пакет з 10 якісних User Stories, ШІ діятиме як Senior Developer. Він згенерує унікальну архітектуру, яка точно відповідає логіці поведінки ваших користувачів.

Захист від непотрібної роботи. Якщо у вашій User Story остання частина («щоб...») звучить не серйозно (наприклад: «я хочу, щоб на задньому тлі грала музика, щоб мені було весело»), ви відразу розумієте, що ця функція безглузда. Музика на тлі дратуватиме 90% користувачів мобільних пристроїв. Історія допомагає відсікти сміття ще до написання першого рядка коду.

Це основа для тестування (QA). Коли в Лабораторній роботі №10 та №11 ви тестуватимете свої віджети та калькулятори через інструменти розробника (DevTools), ви відкриватимете свої User Stories. Тест пройдено успішно лише тоді, коли реальний користувач зміг виконати ланцюжок дій та отримати ту саму вигоду, яка прописана в історії.

Етап 5. Створення текстової карти сайту (Mindmap/Специфікація архітектури)

Для того, щоб в лабораторних роботах №№4-5 почати дизайн та верстку, необхідно мати чітку ієрархію сторінок та блоків.

  1. Попросіть ШІ розробити деревоподібну структуру сайту.
  2. Шаблон Промпту №5 (Генерація карти сайту)
    • "Створи детальну текстову карту (структуру сторінок та інформаційних блоків) для нашого сайту. Розпиши структуру Головної сторінки по блоках (Header, Hero-section, Блок переваг, Блок з інтерактивним віджетом, Форма валідації, Footer). Використовуй дворівневі марковані списки та формат коду Markdown, щоб структуру можна було легко візуалізувати."

Зміст звіту та технічні фінальні артефакти

В результаті виконання роботи студент формує та завантажує на диск (у власну папку) звіт в форматі PDF, що містить:

  1. Титульний лист з зазначенням теми, варіанта (назви бізнес-проєкту).
  2. Розділ 1. Аналітичний бриф (з Grok). 5 головних свіжих трендів для обраної ніші.
  3. Розділ 2. Матриця цільової аудиторії (з Claude). Markdown-таблиця з сегментами, потребами та рішеннями.
  4. Розділ 3. Специфікація функціоналу. Список з щонайменше 7–8 технічних User Stories.
  5. Розділ 4. Архітектура. Текстова карта сайту (структура блоків).
  6. Додаток (Промпт-паспорт/Prompt Log). Скріншоти або логи текстових запитів до ШІ, які демонструють ітераційну роботу (як студент уточнював запити чи виправляв невдалі відповіді моделей).

Критерії оцінювання лабораторної роботи

  • Правильність і повнота Промпт-паспорта. Оцінюється якість промптів. Використання виключно простих запитів типу "напиши мені сайт" зменшує бал. Оцінюється застосування системних ролей, Few-Shot та Chain-of-Thought технік.
  • Глибина аналізу ринку та цільової аудиторії. Наскільки чітко виділено реальні проблеми користувачів, чи враховані сучасні тренди, знайдені за допомогою Grok.
  • Якість та логіка Історій Користувача (User Stories). Відповідність історій користувача стандартам бізнес-аналізу, наявність сценаріїв для головного інтерактивного віджета та форми валідації.
  • Проєктування архітектури. Логічність та повнота структури сайту/блоків для подальшої верстки.
  • Захист роботи та аргументація. Здатність студента усно пояснити, чому обрано саме такі промпти, як відбувалася взаємодія з різними ШІ-асистентами, та довести розуміння згенерованих ШІ артефактів.

Контрольні запитання

  • Блок 1. Промпт-інжиніринг та робота з ШІ (Технічна частина)
    1. Що таке «системний промпт» (або Role Prompting) і чому ви задавали ШІ роль "Senior Business Analyst", а не просто написали звичайний запит?
    2. Поясніть різницю між техніками Zero-Shot та Few-Shot Prompting. Де саме у лабораторній роботі ви використали Few-Shot і навіщо?
    3. Який принцип покладено в основу методу міркувань Ланцюжок думок (Chain-of-Thought)? Як фраза «думай крок за кроком» впливає на логіку відповіді великої мовної моделі?
    4. Чому для збору трендів в реальному часі ви використовували саме ШІ Grok, а не Claude? В чому полягає обмеження стандартних базових LLM щодо актуальної інформації?
    5. Що таке «галюцинація» штучного інтелекту в контексті веб-розробки і як ви, як майбутній фахівець, контролювали чи виправляли результати роботи ШІ у вашому Промпт-паспорті?
  • Блок 2. UX-аналітика та Інформаційна архітектура (Бізнес-частина)
    1. Розкажіть про обрану вами бізнес-тематику. Які два основні сегменти цільової аудиторії виділив ШІ для вашого сайту?
    2. Назвіть одну з ключових потреб вашої цільвої аудиторії, яку виявив Claude, і поясніть, яким саме інтерфейсним рішенням чи блоком на сайті ви плануєте її закрити?
    3. Розшифруйте класичну формулу User Story: з яких трьох обов'язкових частин вона складається?
    4. Яку функцію у вашому списку User Stories виконує остання частина — «щоб...» (so that...)? Чому вона є найважливішою для UI/UX дизайнера?
    5. Покажіть у вашій текстовій карті сайту (Mindmap), де саме ШІ запропонував розмістити головний інтерактивний віджет (калькулятор чи конструктор). Чому він стоїть саме на цьому екрані сторінки, а не, наприклад, в самому низу (Footer)?
    6. Які блоки, окрім Hero-section та калькулятора, ШІ вніс до структури вашої Головної сторінки? Яка логічна роль блоку з відгуками (Testimonials) чи соціальними доказами (Social Proof)?
    7. Як ви плануєте використовувати текстову карту сайту (Sitemap) та блоки сторінки, які ви отримали сьогодні за допомогою ШІ, в наступній лабораторній роботі (ЛР №2)?

Глосарій термінів: UX-аналітика та ШІ-інженерія

Рекомендовано для вивчення перед захистом Лабораторної роботи №1

  • Промпт (Prompt) — вхідна текстова інструкція, запит або команда, яку користувач передає великій мовній моделі (LLM) для отримання конкретного результату (тексту, коду, структури чи графіки).
  • Промпт-інжиніринг (Prompt Engineering) — інженерна дисципліна, що полягає у проектуванні, тестуванні та оптимізації вхідних запитів для ШІ з метою отримання максимально точних, релевантних та передбачуваних відповідей.
  • Системний промпт (System Prompt/Role Prompting) — інструкція найвищого пріоритету, яка задає модель поведінки, роль, обмеження та професійний контекст для ШІ на весь період сесії (наприклад: «Дій як старший бізнес-аналітик в сфері FinTech...»).
  • Нульовий запит (Zero-Shot Prompting) — метод взаємодії з ШІ, за якого моделі дається завдання без надання будь-яких прикладів виконання. Модель спирається виключно на свої базові знання.
  • Запит з прикладами (Few-Shot Prompting) — техніка промпт-інжинірингу, коли в тексті запиту ШІ надається один або кілька готових прикладів (шаблонів) "вхід-вихід". Це радикально підвищує точність форматування (наприклад, для генерації Історії Користувача (User Stories) за суворою формулою).
  • Ланцюжок думок (Chain-of-Thought) — підхід, що змушує ШІ-модель перед видачею фінальної відповіді розділити складне завдання на послідовні логічні кроки (тригерні фрази: «думай крок за кроком», «сформуй внутрішні міркування»). Зменшує ризик логічних помилок.
  • Промпт-паспорт (Prompt Log) — технічний документ або артефакт звіту, куди розробник записує історію своїх ітерацій (змін) промптів, фіксуючи помилки ШІ та вдалі команди, що привели до цільового результату.
  • Галюцинація ШІ (AI Hallucination) — феномен, при якому генеративна модель з високим рівнем упевненості видає логічно зв'язний, але фактично вигаданий, неіснуючий або технічно помилковий контент (вигадані тренди, неіснуючі JS-бібліотеки тощо).

  • Цільова аудиторія (Target Audience) — конкретна група людей, на яку орієнтований веб-ресурс, об'єднана спільними ознаками (вік, стать, професія, рівень доходу) та, найголовніше, спільними потребами чи проблемами.
  • Сегментація цільової аудиторії — розділення загальної аудиторії проєкту на окремі субгрупи (сегменти) за специфікою їхньої поведінки та вимог до продукту (наприклад, сегмент "початківці" та сегмент "професіонали").
  • Потреби користувача (User Pain Points) — реальні проблеми, незручності, страхи, бар'єри або фінансові/часові втрати, з якими стикається клієнт в своєму повсякденному житті, і які повинен вирішити (закрити) ваш веб-сайт.
  • Унікальна ціннісна пропозиція (Unique Value Proposition) — чітка, лаконічна заява на першому екрані сайту, яка пояснює клієнту, яку саме вигоду він отримає, як продукт вирішить його потребу і чому ваш веб-ресурс кращий за конкурентів.
  • Лід (Lead) — потенційний клієнт, який відвідав ваш веб-ресурс і виявив інтерес до продукту, виконавши цільову дію (залишив контакти у формі, скористався калькулятором, підписався на розсилку).
  • Конверсія (Conversion Rate) — відсоткове співвідношення кількості відвідувачів сайту, які виконали на ньому цільову дію (купили, зареєструвалися, прорахували вартість у віджеті), до загальної кількості користувачів за певний час.

  • Історія користувача (User Story) — проста, коротка вимога до функціоналу сайту, що написана мовою кінцевого користувача за суворою UX-формулою: «Як [роль], я хочу [дія], щоб [мета]».
  • Ментальна карта (Mindmap) — інструмент візуального структурування інформації. В лабораторній роботі №1 використовується для деревоподібного відображення взаємозв'язків між блоками та сторінками майбутнього сайту.
  • Карта сайту (Sitemap) — ієрархічна структура всіх сторінок та розділів веб-ресурсу, яка визначає логіку навігації користувача та індексується пошуковими роботами.
  • Інтерактивний віджет (Widget) — самостійний динамічний елемент інтерфейсу сторінки (калькулятор, симулятор, конструктор, фільтр), який реагує на дії користувача в реальному часі без перезавантаження всієї сторінки. Головний інструмент утримання уваги та збору лідів на сучасному лендингу.
Порада на майбутнє від досвідченого розробника:

Коли на захисті викладач запитає вас: «Навіщо ви змусили ШІ писати User Stories?», не кажіть: «Бо так написано в описі лабораторної».

Відповідайте як професіонал: «Я використав техніку Few-Shot Prompting, щоб згенерувати точні User Stories. Це допомогло мені виявити приховану мотивацію користувача й визначити, які саме функції та інтерактивні блоки (наприклад, калькулятор вартості) мають стати ядром архітектури мого сайту для підвищення конверсії».

Робота з ШІ — це мистецтво точних формулювань.
